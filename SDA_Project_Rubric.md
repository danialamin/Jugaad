| Level | Marks | Description |

### Project Requirements Checklist

#### ✅ User Interface (JavaFX/Swing)
- [ ] Functioning user interface in JavaFX or Swing
- [ ] UI elements corresponding to each use case
- [ ] All designed screens implemented
- [ ] All events and interactions working

#### ✅ Business Logic Layer (Application Layer)
- [ ] Must follow OOP principles (encapsulation, inheritance, abstraction, polymorphism)
- [ ] Be prepared to explain usage of OOP principles during evaluation
- [ ] Proper connection between UI and Business Logic layer
- [ ] Well-designed and logical flow

#### ✅ Database Layer
- [ ] Persistence handler must be present
- [ ] Choose appropriate storage method:
  - File-based storage, OR
  - DBMS (Oracle, MySQL, SQL Server, SQLite, etc.)
- [ ] All three layers (UI, BL, DB) must be properly integrated and connected
- [ ] Data properly read, written, updated, and deleted
- [ ] Exception handling implemented
- [ ] Input validation for all user inputs

#### ✅ Use Cases Implementation
- [ ] **All use cases MUST be implemented** (except login/logout)
- [ ] Minimum 4-5 use cases per group member
- [ ] Use cases should be distributed among team members
- [ ] Each use case fully functional

#### ✅ Design to Code Consistency
- [ ] **Classes, attributes, and functions** mentioned in design documents must match implementation
- [ ] **Strict evaluation** of consistency between:
  - Class diagrams and actual code
  - SSD diagrams and UI implementation
  - Use case specifications and functionality
  - Database design and actual schema

#### ✅ Design Patterns
- [ ] Apply **GRASP patterns**:
  - [ ] Controller
  - [ ] Information Expert
  - [ ] Creator
  - [ ] Low Coupling
  - [ ] High Cohesion
  - [ ] Protected Variation
- [ ] Apply **Gang of Four (GoF) patterns** where applicable:
  - Examples: Singleton, Factory, Observer, Strategy, etc.

#### ✅ Self-Evaluation & Documentation
- [ ] Fill the **attached self-evaluation form** (hard copy)
- [ ] Submit hard copy to instructor **before your presentation**
- [ ] Prepare **comprehensive project report** following the provided template
- [ ] Include all required sections with clear documentation

When providing this rubric to an AI for project evaluation:

1. **Share your project structure/code** - The AI needs to assess your implementation against each criterion
2. **Provide screenshots of UI** - For the User Interface criterion
3. **Share your design documents** - SSD, Expanded Use Cases, Class diagrams for consistency evaluation
4. **Provide database schema** - Table definitions and relationships
5. **Share key code snippets** - Especially for OOP principles, architecture, and design patterns
6. **Describe your application** - What does it do? What are the main use cases?

The AI will then map your implementation to each rubric level and assign appropriate marks.
